Abstract

Test-time skill self-improvement is usually treated as a generation problem — search for a better edit to a natural-language skill. We argue it is largely a selection problem. Under a small, noisy validation budget, the standard greedy acceptance gate fuses two very different acts — exploring a candidate and irreversibly committing to it — into one noisy decision, and a scalar score difference cannot distinguish a net-zero change from one that fixes several items while breaking several others. SAGE replaces the scalar gate with a statistical acceptance operator over per-item paired validation evidence: it builds a candidate-versus-incumbent ledger on shared selection items, denoises the measured gain with an exact one-sided sign test, and accounts for regressions before committing — otherwise it abstains and keeps the incumbent. With its safeguards disabled, SAGE recovers the strict SkillOpt baseline gate exactly. This repository is an anonymous, code-only reproduction package: it contains the SAGE implementation, a vendored SkillOpt integration, dry-run launchers for five benchmarks, and zero-API tests, and it intentionally excludes measured results, run outputs, credentials, and paper source.

How SAGE accepts

The method logic is kept small and inspectable. Each stage maps to one module you can read end to end.

1

Paired Evidence

Shared validation item ids become a candidate-versus-incumbent ledger: wins, losses, ties, and incumbent-correct regressions.

sage/evidence.py
2

Exact Denoising

Non-tied paired outcomes pass through a one-sided exact sign test, so sampling noise cannot masquerade as a real gain.

sage/denoise.py
3

Regression-Aware Gate

Denoised gain, loss weighting, and a regression-rate cap decide whether to commit the candidate or abstain.

sage/gate.py
Baseline recovery. Disabling the SAGE safeguards reproduces the strict SkillOpt gate exactly — exercised by tests/test_gate_baseline_recovery.py and configured by configs/skillopt_strict_gate.example.yaml. The bridge from SkillOpt rollout rows to paired scores lives in sage/skillopt_bridge.py.
